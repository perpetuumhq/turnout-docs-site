JOURNEY 1: OWNER PRE-LAUNCH (turnout.club → turnout.so → qualification → OG)

Owner discovers Turnout via Instagram (@turnout.club), lands on turnout.so, pre-orders, qualifies via automated scoring, and enters launch cohort with OG status if score ≥70.

Screen 1.1: turnout.so Landing Page

What the owner sees:

  • Hero section: "Your venue deserves a landing page that works" + demo video (15s) of a Turnout page in action
  • Social proof: "Already live with [N] venues in Delhi, Bangalore, Mumbai" + 2-3 testimonial cards (venue name, category, result quote)
  • 3-panel feature breakdown: "Real Photos Only → AI Composition → WhatsApp Concierge" with icons
  • Pricing visible: "Pre-order for your city: Rs 4,999/month OR Rs 12,999 annual"
  • CTA button: "Claim Your Spot" (primary) + "See Sample Pages" (secondary)
  • Bottom section: FAQ (5 common questions: "How fast?", "Do you use my photos?", "What if I'm not on Instagram?", "Can I cancel?", "Who has early access?")

Interactive elements:

  • Demo video plays inline, with skip option
  • "See Sample Pages" button opens modal with 4 sample pages (Sports, Brunch, Fitness, Dining) — user can swipe through
  • "Claim Your Spot" → scrolls to form

Algorithm: Static landing. No personalization yet. Forms measure: video view time, CTA button click, form submission.

Edge cases: Mobile-optimized. If video doesn't load, show fallback static image carousel. If user already submitted form in past 7 days, show "Your application is being reviewed" instead of CTA.

Screen 1.2: Pre-Order Form (turnout.so/pre-order)

What the owner sees:

  • Form fields (required marked with *):
    • Venue name *
    • Category dropdown (Restaurant, Cafe, Sports, Fitness, Wellness, Nightlife, Event Space) *
    • City dropdown (Delhi, Bangalore, Mumbai) *
    • Owner email * (auto-prefill if logged in)
    • Owner phone * (WhatsApp-enabled number)
    • Venue address *
    • Instagram handle (optional) — "We'll scrape your public posts to build your profile"
    • Website URL (optional) — "We'll pull your menu, photos, and hours"
    • Google Places URL (optional) — "Link to your Google Business listing"
  • Consent checkboxes (required):
    • ☐ "I consent to Turnout analyzing my Instagram (public posts, follower count, engagement)" *
    • ☐ "I consent to Turnout accessing my website (menu, photos, hours, pricing)" *
    • ☐ "I consent to Turnout contacting me via WhatsApp and email" *
    • ☐ "I understand Turnout will ask me 5-8 discovery questions next" *
  • Submit button: "Start Pre-Order" (disabled until all * fields filled)
  • Bottom: "We'll reach out in 2 hours with next steps"

Interactive elements:

  • All fields have client-side validation (email format, phone format, min 3 chars for name)
  • Instagram handle field includes autocomplete for verified accounts (if exists)
  • Clicking consent checkbox reveals tooltip explaining what data Turnout accesses and how
  • Submit button shows loading spinner and "Submitting..." during API call

Algorithm: Form submits to /api/preorder endpoint. Creates owner profile record. Immediately triggers qualification engine in background (scrapes Instagram, website, Google). Sends confirmation email + WhatsApp message with next steps link.

Edge cases: If Instagram handle invalid or private, skip IG scrape and note "IG data unavailable" in profile. If website unreachable, skip website scrape. If phone number not WhatsApp-enabled, send SMS confirmation + email. If owner submits duplicate (same email), show "You already pre-ordered. Check your email for status."

Screen 1.3: Journey Questions (5-8 structured questions)

What the owner sees:

Owner receives WhatsApp message: "Hi [Venue]! We're excited to help. Please answer 5 quick questions so we can recommend the perfect template for you."

Questions (presented one-by-one in WA, or as a form link to turnout.so/journey/[owner_id]):

  1. How long has your venue been operating? → Single choice: "0-6 months" / "6-12 months" / "1-3 years" / "3+ years"
  2. What's your biggest current challenge? → Single choice: "Low footfall during off-peak hours" / "Awareness among target audience" / "Difficulty coordinating group bookings" / "Price sensitivity from customers" / "Not sure yet"
  3. What's your venue's primary use case? → Single choice: "Walk-in (customers just show up)" / "Reservation-based (customers book ahead)" / "Slot/class-based (timed sessions)" / "Both walk-in + reservation"
  4. How many real photos do you have of your space? → Single choice: "1-5" / "5-15" / "15-30" / "30+"
  5. Are you comfortable with pricing experiments? → Single choice: "Yes, optimize for revenue" / "Yes, optimize for volume" / "Prefer to keep pricing fixed" / "Unsure"
  6. [Optional] What's your current monthly marketing spend? → Text input: "Rs X" or "Not tracked"
  7. [Optional] Are you on any other booking platforms (Zomato, Dineout, etc)? → Checkbox group: IG / Website only / Zomato / Dineout / EazyDiner / Other
  8. [If restaurant/cafe] Do you have a digital menu available? → Yes / No / Uploading soon

Interactive elements:

  • In WA: Each question sent as interactive button list (if WA Flow API available), or free-text with user response parsing
  • In form: Each question is a radio group or dropdown
  • Progress indicator: "Q3/5" at top of form
  • Can go back to edit previous answer

Algorithm: Qualification Scoring Engine (runs on form submit). Inputs to scoring:

  • Content Readiness: "How many photos?" + IG scrape result (photo count, quality) + website scrape (menu, gallery) → 0-25 points
  • Commercial Readiness: Venue category + "reservation vs walk-in" answer + pricing visibility from website/Zomato → 0-25 points
  • Growth Signal: "Biggest challenge" answer + venue age + IG engagement trend + owner willingness to experiment → 0-25 points
  • Theme-City Fit: Category + city + owner's booking model (e.g., "Sports + Delhi + Reservation" = high fit) → 0-25 points

Qualification Score Logic:

Content Readiness (0-25):

  • 1-5 photos = 5 pts | 5-15 = 10 pts | 15-30 = 18 pts | 30+ = 25 pts
  • Bonus: +5 if IG engagement rate > city median | +5 if website gallery has 10+ distinct images

Commercial Readiness (0-25):

  • Pricing visible on website = 10 pts | Reservation system active = 8 pts | Zomato/Dineout presence = 7 pts
  • Venue age 1+ years = 5 pts | Category in Wave 1 (Sports, Brunch, Wellness, Dining) = 10 pts

Growth Signal (0-25):

  • "Off-peak footfall" challenge = 10 pts (high motivation) | "Awareness" challenge = 8 pts | "Group coordination" = 7 pts
  • Willing to experiment (pricing/volume) = 10 pts | Prefer fixed pricing = 3 pts
  • IG posting frequency > 2x/week = 5 pts bonus

Theme-City Fit (0-25):

  • Category match to Wave 1 heatmap + city priority. Example: "Pickleball + Delhi" = 25 pts. "Cloud kitchen + Mumbai" = 8 pts.

Total Score Threshold: Launch Cohort = 55+. OG Status = 70+.

Edge cases: If owner skips questions, use defaults from pre-order form (category, IG handle) to estimate. If score < 55, system still qualifies but shows "improvement roadmap." If duplicate answer detection (copy-paste), flag for manual review.

Screen 1.4: Qualification Result (Score & OG Status)

What the owner sees:

Owner receives WhatsApp message or email with result. If qualified:

  • Score display: "You scored 72/100 — Congratulations, you're a Turnout OG!" (big badge with confetti animation if Web)
  • Score breakdown: Bar chart showing 4 sub-scores (Content: 22/25, Commercial: 19/25, Growth: 18/25, Theme-City: 13/25)
  • What this means: "As a Turnout OG, you get: Priority onboarding (this week), Access to exclusive community, Free content shoot (photos + video), Early feature access"
  • Next steps: "Click below to schedule your content shoot or start onboarding immediately" (2 buttons)

If not qualified (score < 55):

  • Result: "Score: 48/100 — You're close! Let's get you launch-ready."
  • Improvement roadmap:
    • "Your content is strong (20/25). No action needed."
    • "Your booking setup needs work (8/25). Action: Add pricing and hours to your website (estimated time: 30 min)."
    • "Your growth signal is uncertain (12/25). Action: Post 5 more behind-the-scenes photos on IG this week."
  • Requalify button: "Re-take the journey questions in 7 days to improve your score"
  • Alternative CTA: "Still interested? Let's talk to our team" → book call with founder

Interactive elements:

  • Score breakdown bars are interactive (hover shows detail)
  • Improvement actions are clickable (e.g., "Add pricing to website" links to tutorial)
  • CTA buttons: "Schedule Content Shoot" or "Start Onboarding Now"

Algorithm: Qualification score persists in owner profile. Score triggers template recommendation confidence level in Journey 2.

Edge cases: If owner was previously not qualified but requalifies, celebrate the progress. If score drops on re-qualification (venue removed photos from IG), notify owner proactively.

What the owner sees:

  • Enriched Venue Data Card (read-only, with edit link):

    • Location: "[Address from Google]" (edit link)
    • Hours: "[From Google/website]" (edit link)
    • Average rating: "4.2⭐ from 87 Google reviews" (if exists)
    • Price range: "[Rs XXX-YYY per person]" (if detected from website/Zomato)
    • Vibe tags (auto-generated from IG captions): "Casual • Instagram-worthy • Group-friendly" (edit link)
  • Trust Meter (0-100): "Trust Score: 78/100 — Your data is strong. Most sections of your page will be Tier 1." (Visual bar with explanation)
  • Content Summary:
    • "✓ Instagram: 2,450 followers, 4.2% engagement, 24 recent posts (excellent)"
    • "✓ Website: Menu, 12 photos, hours visible (strong)"
    • "✓ Google Places: 87 reviews, last updated 3 days ago (fresh)"
    • "⚠ Missing: Team bios, chef credentials. This is optional but helps build trust."
  • CTA: "Great! Let's build your page. Ready?" → "Next"

Interactive elements:

  • All "edit link" fields are clickable and open inline forms
  • Trust score bar is interactive (hover shows breakdown by component)
  • Vibe tags show edit icon on hover

Algorithm: Venue Profiler (Skill 1) outputs enriched profile. Trust score calculated from 6 components (Media Richness, Review Credibility, Operational Proof, Credential Depth, Revenue Signal, Owner Engagement). Confidence = qualification score + freshness of data.

Prompts:

# System: Vibe Tag Generator You are analyzing Instagram captions and Google reviews to extract the emotional/cultural vibe of a venue. Input: - Venue name: {{venue_name}} - Category: {{category}} - Recent IG captions (10 most recent): {{ig_captions_list}} - Google reviews (top 5 by helpfulness): {{google_reviews_list}} Task: Generate 3-5 vibe tags that capture the *feeling* of this venue, not just its function. Format: Return as JSON: { "vibe_tags": ["tag1", "tag2", "tag3"], "explanation": "Why these tags matter for this venue" } Rules: - Use common cultural motifs (e.g., "Influencer-approved", "Gram-worthy", "Local favorite", "Upscale escape", "Student-friendly", "Family-safe", "Wellness-focused") - Avoid generic tags ("Nice place", "Good food") - If captions/reviews mention specific themes (e.g., "bring your yoga mat", "sunset views", "group-friendly"), prioritize those

Edge cases: If IG private or no data available, skip IG enrichment and note "IG data unavailable." If venue just opened (< 7 days), Trust score capped at 45 but show "New venue? We'll learn as you grow." If Trust < 50, highlight "Content Gap" section and ask owner to upload photos.

Screen 2.2: The −1 Journey Page ("Venues like yours face these problems")

What the owner sees:

  • Hero: "Venues like yours typically face 3-4 specific challenges. Let's solve them."
  • Problem Cards (AI-generated, personalized by category + city + challenge from qualification):
  • Each card has:
    • Problem title (e.g., "Off-peak slots sit empty")
    • Why it happens (1-2 sentences, specific to venue category/city)
    • Impact: "This costs you ~Rs X/month in lost revenue" (estimated based on category benchmarks)
    • How Turnout solves it: "WhatsApp concierge converts hesitant leads. Pricing experiments find demand."
    • Evidence: "Similar venues saw 23% footfall lift in month 1"
  • Example problems (by category/city):
    • Brunch cafe in Bangalore: "Weekday mornings are slow. Customers don't know about your new filter coffee blend."
    • Pickleball court in Delhi: "Off-peak slots (Tue-Thu 2-5pm) are empty. Beginners don't feel confident booking. Corporate groups don't know about your team rates."
    • Yoga studio in Mumbai: "New students see your schedule and don't book because they're nervous. Group class coordination is chaotic on WhatsApp."
  • CTA: "These are the problems we solve. Ready to build your page?" → "Next"

Interactive elements:

  • Problem cards have "Learn more" link (expands to show case study or FAQ)
  • Evidence section shows confidence level (e.g., "Based on 47 similar venues")

Algorithm: Problems Database (seeded from market research) × Venue Profile × City-Theme Heatmap. Retrieves top 3-4 problems most relevant to this venue's category, city, and stated challenge (from Journey 1 questions).

Prompts:

# System: Problems Generator (−1 Page) You are a venue operations expert helping owners understand their category-specific challenges. Input: - Venue name: {{venue_name}} - Category: {{category}} - City: {{city}} - Stated challenge (from qualification): {{owner_challenge}} - Venue age: {{venue_age}} - Current footfall tier: {{footfall_tier}} (estimated from Google/Zomato) Task: Generate 3-4 problem statements specific to this venue. Each problem should: 1. Feel real and relatable (not generic) 2. Have quantified impact (estimated Rs loss per month) 3. Show how Turnout solves it 4. Reference evidence from similar venues Format: { "problems": [ { "title": "Problem title", "why": "Why this happens specifically at {{venue_name}}", "impact": "This costs ~Rs X/month in lost revenue", "solution": "How Turnout solves it", "evidence": "Similar venues saw X% improvement" }, ... ] } Rules: - Draw from these categories-specific patterns: * Brunch cafes: Weekday awareness gap, group coordination, walk-in discovery * Sports (pickleball): Off-peak booking anxiety, beginner confidence, group pricing * Fitness: Beginner hesitation, class coordination, package upsells * Fine dining: Group coordination, occasion framing, deposit communication - Use city-specific language (e.g., "South Delhi crowd wants Instagram-worthy spots", "Bangalore early-birds want weekday deals") - If {{owner_challenge}} is known, amplify that problem in the list

Edge cases: If venue too new to have reliable data, use category averages. If owner stated "Not sure" for challenge, show 4 generic problems for category. If venue is already high-performing (Google rating 4.5+), reframe problems as "optimization opportunities" not "problems."

What the owner sees:

  • Hero: "Choose your page template. Based on your venue, we recommend [Template Name]."
  • Recommended Template Card (highlighted with accent color border):

    • "Turnout recommended based on your category and booking model"

    • Features: "• Court/slot picker • Group split-pay • Beginner reassurance • Pack options"
    • Preview button: "See a demo" → opens side-by-side comparison showing how the page looks for a sample venue in same category
    • CTA: "Use this template"
  • Alternative Template Cards (2-3 options): Same structure, but without "recommended" tag. Example alternatives: "Reservation-First" (if venue fits multiple categories), "Premium Dining" (if upscale fine dining).
  • Preview feature: When user clicks "See a demo" on any template, show side panel with:
    • Mobile phone mockup (iPhone 12 Pro size)
    • Actual page from similar real venue in same category/city
    • Scroll-through animation showing all 5 screens of customer flow
    • Overlay explaining each section: "This hero image is from a venue like yours"

Interactive elements:

  • Template card hover effect (slight scale up, shadow increase)
  • "See a demo" button opens full-screen modal
  • Demo modal has close button + "Next" to see next template's demo
  • Selecting template highlights card border and shows checkmark

Algorithm: recommend_template() function (outputs in Trust × Taste Graph). Uses Trust score (high = more aggressive Tier 1 creativity) and Taste profile (category × city × vibe tags) to select best template. Confidence score returned (75-100 = high confidence, "Use this"; 50-74 = medium, show alternatives; <50 = "Choose what feels right").

Prompts:

# System: Template Recommendation Explanation You are the product recommending a page template to a venue owner based on their profile. Input: - Venue name: {{venue_name}} - Category: {{category}} - City: {{city}} - Qualification score: {{qual_score}}/100 - Trust score: {{trust_score}}/100 - Taste profile: {{taste_vector}} (e.g., "sports-social, group-focused, outdoor") - Owner's stated challenge: {{challenge}} Task: Recommend the best template and explain why in 1-2 sentences. Output: { "recommended_template": "Template name", "confidence": 85, "why": "Why this template is best for them", "features_highlight": "Why these specific features solve their challenge" } Rules: - Use the taste_vector to match template. If "sports-social + group-focused", recommend Sports-Social template. - If trust_score < 50, recommend the most lenient template with minimal AI composition - If challenge == "off-peak footfall", recommend template with strong daypart/offer features - If owner is repeat, recommend their previous template (for consistency) unless they changed challenge

Edge cases: If multiple templates equally good (confidence 65-75), show 3 options with equal prominence. If trust score too low to support any template confidently, show "Let's improve your data first" and return to Screen 2.1. If owner selects non-recommended template, ask "Are you sure?" with explanation of why recommended one might be better.

Edge cases: If owner selects incompatible primitives (e.g., "Waitlist" for a brunch cafe with walk-in focus), show warning: "This primitive may not match your venue model. Sure?" If owner selects many add-ons (5+), show cost summary and ask "Does this fit your budget?" If owner deselects all recommended add-ons, nudge: "We recommend Group Split-Pay for your venue. Sure you don't want it?"

Algorithm: Concierge Flavor Mapping (from Algorithm × UX Contract). Stores concierge_tone + knowledge_facts in owner profile. These feed into WA conversation system prompt (Journey 5).

Prompts:

# System: Concierge First Message Generator You are personalizing the first message the WA concierge will send to a customer. Input: - Venue name: {{venue_name}} - Category: {{category}} - Tone: {{tone}} (friendly, professional, enthusiastic) - Knowledge facts: {{knowledge_facts}} Task: Generate a natural, warm first message that: 1. Acknowledges the customer landed on the venue page 2. Introduces the concierge 3. Offers help 4. Edge cases: If owner leaves knowledge_facts empty, use category defaults (e.g., "Sports-Social: 'Beginner-friendly, no gear rental needed'"). If tone selection seems mismatched to category (e.g., "Casual" for upscale fine dining), show nudge but allow. If owner enters off-brand knowledge (e.g., "We serve alcohol" for wellness studio), flag for review.

Screen 2.6: Final Review + Page Creation

What the owner sees:

  • Hero: "Review everything before we build your page."
  • Review Summary (read-only cards, each with "Edit" link):
    • Venue Profile: Name, category, location, hours, rating
    • Template: Template name + "Preview"
    • Primitives: Selected features (Reservation, Group Split-Pay, etc.) + monthly cost

  • CTAs:
    • Primary: "Create My Page Now" (triggers page generation API)
    • Secondary: "I need to edit something" (each card's "Edit" link goes back to relevant screen)

Interactive elements:

  • Each summary card has "Edit" link (goes back to relevant screen in Journey 2)
  • Create button shows spinner and "Generating..." during API call
  • On success: "Your page is being created! Check your email in 2-3 hours." + journey to Journey 3 (owner-as-customer test)

Algorithm: Triggers Page Composition Engine (Skill 3) with inputs: template selection, venue profile, primitives config, concierge flavor. Generates page layout, content, and configuration. Page status stored as "draft" until owner approves (Journey 3).

Edge cases: If page generation fails (e.g., insufficient photo data), show "Let's improve your page" and return to Screen 2.1 to collect more photos. If generation takes >30 min, send progress email + WA notification. If owner tries to edit 5+ times, show message: "Let's get a human review. Chat with our team?" and link to concierge chat.

JOURNEY 3: OWNER-AS-CUSTOMER TEST + SHAREABLE ARTIFACTS

Owner experiences their page as a customer would, provides feedback, approves or requests edits, then generates shareable preview cards and OG badge for promotion.

Screen 3.1: Customer Mode Toggle ("Experience Your Page")

What the owner sees:
