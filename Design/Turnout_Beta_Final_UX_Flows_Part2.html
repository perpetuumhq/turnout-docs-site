      • Concierge tone/knowledge: Radio + text (as in Journey 2.5)
      • Testimonials: Select from real reviews or add custom
  • Action buttons: "Save Draft" (local storage, not published) / "Preview on Phone" (full-screen phone preview) / "Publish Changes" (live update, versioned)

Edit Workflow:

  1. Owner clicks "Edit Page"
  2. System loads current page configuration into edit form
  3. Owner makes changes (fields auto-save as "draft")
  4. Owner clicks "Preview on Phone" (side panel shows new version)
  5. Owner satisfied → "Publish Changes"
  6. System creates new page version (v1.0 → v1.1). Tracks what changed.
  7. Customer re-visiting page sees new version (transparent update)

Version History + Rollback:

  • Every published version saved with timestamp + owner notes
  • Owner can click "View Previous Version" to see v1.0, v1.1, etc.
  • "Rollback to V1.0" button available (rare case: accidental bad edit)
  • Analytics tied to versions (can compare v1.0 CTR vs v1.1 CTR)

Algorithm: Page configuration stored as JSON (templates + fields). Edit form is dynamic (fields generated from page schema). On publish, new version created, analytics event logged, old version retained.

Edge cases: If owner tries to remove all hero images, warn: "Every page needs a hero image. Choose one to continue." If owner publishes page with no pricing data, warn: "Pricing is required. Set it before publishing."
